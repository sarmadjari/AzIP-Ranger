Inspection & Firewalls

North-South tier HA (ILB)
East-West tier HA (ILB)

Each tier holds an ordered chain of NVA groups: route tables send traffic to the entry group's VIP (.100); each group forwards to the next hop (.101, .102…) and SNATs to its internal NICs, so every chain segment stays flow-symmetric. VMs: 1 = single instance (NIC next hop — SPOF), 2–3 = active-active behind the group ILB, each NVA individually nameable. VMSS: autoscale min–max, dynamic instance IPs, the VIP is the anchor. Reorder with ↑↓. Full inventory + chain contract in the IP Plan tab.

Position in chain

Dual N-S: workloads → Azure Firewall → NS NVAs → Internet (RT-AzureFirewallSubnet with 0/0 → NS ILB). Dual E-W: spokes/on-prem → Azure Firewall → EW NVAs → destination. Single-tier: everything → Azure Firewall → the NVA tier — no tier choice needed.

Azure Firewall is platform-HA (availability zones) — no ILB or instance count needed. East-west SNAT requires Private IP ranges = 255.255.255.255/32 on the Firewall Policy ⓘ
